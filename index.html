Huffman Compression Concept Guide

Algorithmic Process

Huffman Coding uses a greedy approach to build a prefix binary tree. The steps are:

  1. Count frequencies of all unique bytes in the input stream.
  2. Initialize leaf nodes for each byte and insert them into a Min-Heap.
  3. Extract the two nodes with the lowest frequencies.
  4. Merge them under a new parent node, summing their frequencies.
  5. Insert the parent node back into the Min-Heap.
  6. Repeat steps 3–5 until only one node (the root) remains.
  7. Traverse the tree to assign paths: left = 0, right = 1.

Prefix-Free Coding

Because leaf nodes are the only nodes that store character bytes, no character's code can be a prefix of another code. This ensures a bitstream can be parsed from start to finish without delimiters, making it uniquely decodable.

Binary Packing Header Format

A compressed .huff file is structured into three parts:

Header Size (4 Bytes)
JSON Metadata (Frequency Table & Padding Count)
Padded Bitstream (Byte-Packed Bits)

The first 4 bytes are unpacked as a big-endian uint32, telling the decompressor exactly how many bytes to read to parse the header containing the metadata.

CompressIQ — DSA Algorithms

Interactive Algorithm Tester

01

Frequency Count

O(n) Time O(k) Space

Single linear pass over all bytes to build a dictionary mapping each unique byte to its occurrence count. This is Step 1 of the Huffman pipeline.

Run the demo to see results.
02

Insertion Sort

O(n²) Time O(1) Space

In-place stable sort. Efficient for very small arrays. Used as a baseline comparison to demonstrate why Merge Sort is preferred for codebook sorting.

Run the demo to see results.
03

Merge Sort

O(n log n) Time O(n) Space

Divide-and-conquer sorting. Splits the frequency list in halves recursively, then merges them in sorted order. Guarantees O(n log n) in all cases.

Run the demo to see results.
04

Binary Search

O(log n) Time O(1) Space

Searches a sorted codebook for a target byte value by halving the search window each step. Used to locate symbols in pre-sorted codebook arrays.

Run the demo to see results.
05

Min-Heap (Priority Queue)

O(log n) Insert/Extract O(n) Space

Custom array-based binary tree. Always keeps the minimum-frequency node at index 0. Parent index: ⌊(i-1)/2⌋. Left child: 2i+1. Right child: 2i+2.

Run the demo to see results.
06

Greedy Tree Build

O(n log n) Total O(n) Space

Core Huffman algorithm. Repeatedly extracts the two lowest-frequency nodes, merges them under a new parent, and re-inserts until only the root remains.

Run the demo to see results.
07

BFS Tree Traversal

O(n) Time O(w) Space

Breadth-First (level-order) traversal using a queue. Visits the root, then all level-1 nodes, then level-2, etc. Reveals how common characters sit near the root.

Run the demo to see results.
08

DFS Tree Traversal

O(n) Time O(h) Space

Pre-order depth-first traversal (Root→Left→Right). At each leaf, the accumulated path of 0s and 1s IS the Huffman code for that character.

Run the demo to see results.
09

Shannon Entropy

O(n) Time O(k) Space

H(X) = −Σ P(x)·log₂(P(x)). Calculates the theoretical minimum bits-per-symbol. Huffman coding always achieves within 1 bit of this limit.

Run the demo to see results.
10

Bit Packing

O(n) Time O(n/8) Space

Groups 8 consecutive bits into one physical byte using integer bit-shifting. Without this step, writing '0'/'1' as text characters would make files 8× LARGER.

Run the demo to see results.
